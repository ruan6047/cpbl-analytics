## 背景

現行能力卡守備軸 `RF = (PO+A)/G` 衡量「經手多少球」而非守備價值；改分母只修正時間基準，
不改變衡量對象（見 [`UX-ABILITY-FIELD1_PHASE1.md`](../research/UX-ABILITY-FIELD1_PHASE1.md)，
該卡 Phase 2 已依需求方裁決不執行）。

國外對「無座標資料」的處境有成熟解法：[Total Zone](https://www.baseball-reference.com/about/total_zone.shtml)
只需逐打席的「誰處理的」，估算超越期望的出局數。我們的 `game_livelog` 2018–2026 有約
**13.6 萬筆**可抽出守位的事件（逐球追蹤僅 5,998 筆），且可跨季累積以滿足
[UZR 文件](https://library.fangraphs.com/defense/uzr/)明訂的「三年才可下結論」門檻。

## 研究問題（本卡要回答的）

- [x] **文字解析涵蓋率**：**95.31%**（138,370／145,178 擊球入場），九季穩定 94.88–96.26% 無斷層。殘餘 4.69% **有系統性偏誤**：犧牲觸擊 2,159 筆全數落空（「擊出球(觸擊)」無方向詞），系統性低估投／捕／一／三壘手機會。詳 [報告](../research/ML_FIELD_TZ1_FEASIBILITY.md) §1。
- [x] **期望值模型怎麼建**：**naive 查表法否決；內野迴歸式實務不可行、唯外野可行**（iteration 3 依查核 Critical Finding 收斂）。`content` 的「方向」是處理結果非落點（內野滾地球出局率 89.3% vs 滾出外野 0.2–0.5%），naive 查表法屬循環論證。迴歸式雖使參數**數學可識別**（配對網絡實測單一連通元件），但實務兩關卡無解：(a) 隊內共現 60–80%＋跨隊真橋樑稀疏→估不準；(b) 方向桶歸屬由守備成敗決定的**分類洩漏**，collapse 後仍破不掉循環。詳 §2.3。
- [x] **跨季累積與年代校正**：**外野空中球須合併高飛＋平飛、以年份 fixed effect 統一吸收**（iteration 3 依查核 Critical Finding 2 收斂）。平飛球嚴重漂移（11.45%→1.95%，接殺平飛球被改記高飛球，使高飛球「平坦」為假象）；分桶各自校正不足，合併後出局率九季 49.5%→57.8% 仍漂移，故須帶年份控制。詳 §3。
- [x] **驗證方案**：已執行場次數對帳（逐年與 spec 基線 §3 完全一致）與每場打席數對照官方 PA（+1.37% 過度切分不污染出局率，查核 Finding 3 佐證）。若實作建議三階：出局守恆 ±3%、與官方守備率中度正相關、金手套僅作弱訊號不可當 ground truth。詳 §4。
- [x] **規模與風險評估**：發現未預見前置——**無逐場守備陣容**（`game_detail` 無先發、`fielding_innings` 僅球季層級），須由打者列 `defend_station_code` ＋換防事件重建（外野路線的關鍵前置，不需新爬蟲）。外野三季累積約 885 次機會、精度 ±14 個機會。核心風險為內野分類洩漏＋共線性（實務 NO-GO）與外野空中球判定漂移（須年份控制）。詳 §5。

## 研究結論（執行者，iteration 3）

- **naive 期望值查表法 TZ：NO-GO** —— 無座標下查表等於用結果預測結果（循環論證＋單顆球歸屬欠定）。
- **內野迴歸式 TZ：極高風險 → 實務 NO-GO** —— 參數數學可識別但估不準（共現 60–80%＋稀疏橋樑）＋方向桶分類洩漏無法消除。**不建議投入開發資源。**
- **外野空中球範圍指標：唯一實質可行（條件式 GO）** —— 全具名、無分類洩漏；須合併高飛＋平飛以年份 fixed effect 吸收漂移＋逐局陣容重建；三年門檻、精度 ±14 機會。屬另卡並須需求方核可。
- **不建議以 TZ 取代能力卡守備軸**：唯一可行的外野指標填不滿內野，跨守位不可比。短期零風險緩解＝多季滾動標示（spec 基線方向三）。
- 一句話：內野在現有資料下**沒有**可信的個人守備指標（無獨立落點座標）；能做的是外野空中球範圍與即時多季標示。四路徑詳報告 §6.1。

> 結論三輪演變：iteration 1「一律 NO-GO」（過悲觀）→ iteration 2「迴歸式條件式 GO」（過樂觀）→ iteration 3「內野實務 NO-GO、唯外野條件式 GO」（校準）。每輪皆由跨家族查核推動；修訂紀錄見報告 §7。
