## 查核細項說明

### 1. BBox 文字框相交檢測（紅線驗收）
採用 Playwright 對走查頁之六大情境（3守位、5守位、9守位、超長副標、副標缺值、全空）及三名真實球員進行自動化 `getBBox()` 檢測：
- **余德龍 (`0000003563`)**（3守位，原缺陷「45 局」↔「三壘」重疊）：**0 相交**。
- **高捷 (`0000007091`)**（5守位，原缺陷「12 場」↔「一壘」重疊）：**0 相交**。
- **劉俊豪 (`0000006743`)**（6守位）：**0 相交**。
- **375px 行動版與 1280px 桌機版**：檢測結果均為 **0 相交**，且**無水平溢出**（`docScrollW === innerWidth`）。
- **先驗相交偵測反證**：腳本於測試中強行將一個 text 移至另一個 text 位置，偵測器立即正確回報相交（`左外野 ↔ 1234 局`），還原後歸零，證實相交檢測有效且非空轉。

### 2. 共用元件設計與轉接層
- **元件位置**：`web/src/components/field-diagram.tsx`，樣式使用轉播風制式排版，無真實球場座標，未使用格位以虛線與淡色背景呈現，可讀性高。
- **Props 界面**：`cells: FieldCells`，為「守位碼 → {main, sub}」映射，不假設資料來源。此設計完全能容納未來賽況頁（先發守位＋守備異動）之用途。
- **轉接層**：英文守位對應、局數/場數格式化（`fieldingSub`、`fieldingCells`）確實留在球員頁側的 `web/src/app/players/[id]/fielding-metrics.ts`，未滲入共用元件。
- **副標缺值**：副標缺值時元件不渲染對應的 `<text>`，不留空殼，無資料格不吃副標。

### 3. 同守位多列累加累計（fieldingCells）
- 實測季中轉隊者（同守位多列）的累計邏輯。
- **缺陷 Revert 實測**：臨時將 `fieldingCells` 修改為會覆蓋舊資料的缺陷版，跑單元測試成功跑紅並報出預期的 `'7 局' !== '87 局'` 錯誤；還原為 Map 累加版後測試恢復綠燈，證實此防禦性回歸測試確實有效。

### 4. 估寬與截斷魯棒性
- **文字寬度估計 (`textWidth`)**：半形 0.6em、全形 1.0em。此算法在真實渲染中（如大寫英文字母與 CJK 混排）具有安全緩衝，且經 getBBox 驗證，超長副標均能被 `fitText` 正確截斷並加上 `…`，最壞情況下仍不超出格子邊界。
- **Aria-label 語音描述**：語音讀出未截斷的原文（例如超長副標 `戴培峰（第 7 局換上）`），不受格子寬度限制，確保無障礙體驗取得完整資訊。
