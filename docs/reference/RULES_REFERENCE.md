## 2. 各文件結構

### 棒球規則（OBR 章節）
1.00 比賽之目的 / 2.00 球場 / 3.00 裝備制服 / 4.00 比賽準備 / 5.00 比賽進行 /
6.00 不正確 Play 與不當行為 / 7.00 比賽結束 / 8.00 裁判員 / **9.00 記錄規則**（p.157–）。
卷首含當年度修正對照（2026：5.02(c) 罰則等）。

### 規則補述（六章）
一、競賽規程（1.01–1.15：延長賽/正式比賽/因雨/保留/登錄違規/暫停/**投球計時**/抗議/驅逐/裝備）
二、投手篇（2.01–2.09：姿勢/牽制/犯規/狙擊）
三、妨礙篇（3.01–3.13）
四、其他特殊狀況 五、比賽場地規則概況 六、裁決可否更判

### 聯盟規章（組織章程 7 章 + 辦法 11 章）
第 8 章 註冊辦法（**第 6 條外籍球員人數**）/ **第 9 章 聯盟競賽規程**（第 12 條登錄上場、
第 37–38 條延長賽與正式比賽裁定）/ 第 10–11 章 熱身賽・明星賽 / **第 12 章 季後賽及總冠軍賽**
（第 66–68 條）/ 第 13–14 章 國際賽・邀請賽 / **第 15 章 年度獎勵**（第 78–82 條，含第 80 條
票選遴選標準、100 救援等里程碑表揚）/ 第 16 章 新人選拔（含第 86 條旅外球員）/
**第 17 章 外籍球員管理** / 第 18 章 合約讓渡（第 104 條選手登錄）。
⚠️ 組織章程與辦法的「條」各自編號（組織章程也有第 37 條＝會計年度），引用時須帶章別。

## 3. 專案資料語意對照（分析重點）

| 專案資料/邏輯 | 權威條文 | 要點（實作時查原文） |
|---|---|---|
| `games.delay_kind`/`orig_date`（延賽保留，記憶 `game-calendar-and-names`） | 補述 1.02–1.04；規章第 9 章第 38 條 | 未賽完 1 局＝重賽不算；2 局–未滿 5 局＝**保留擇期續賽**（紀錄認定另計）；保留比賽恢復時上場球員限制見 1.04 |
| 延長賽/和局語意（outcome 模型 label、戰績和局） | 補述 1.01；規則 7.01(b)(2) | 例行賽 10 局起**突破僵局制**、至多 12 局；補賽雙重賽第一場不延長；**天母 22:00 宵禁**特例；季後/總冠軍（C/E）必須完整九局、不採裁定 |
| 救援重建（記憶 `special-records-feature`：無 save 旗標靠 scoreboard 重建）與**中繼救援判定 TODO** | **規則 9.19**（p.182）；中繼點 9.20 前後 | 救援四要件：勝隊最後一任投手、非勝利投手、≥1/3 局、且（領先≤3 分投滿 1 局／追平分在壘上或打擊區／投滿 3 局）——重建演算法必須逐要件對齊 |

## 4. 使用方式

```bash
# 查條文（txt 含分頁標記，可還原 PDF 頁碼）
grep -n "救援成功" docs/reference/棒球規則.txt
# 更新（規則年度改版時重下載三份 PDF + 重抽 txt）
uv run --with pypdf python - <<'PY'
from pypdf import PdfReader
for f in ('聯盟規章', '裁判執法手冊規則補述', '棒球規則'):
    r = PdfReader(f'docs/reference/pdf/{f}.pdf')
    open(f'docs/reference/{f}.txt', 'w').write(''.join(
        f'\n===== 頁 {i} =====\n' + (p.extract_text() or '') for i, p in enumerate(r.pages, 1)))
PY
```
