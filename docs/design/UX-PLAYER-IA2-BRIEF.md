# UX-PLAYER-IA2 球員頁 IA 修訂：role 拆標籤＋守備獨立層 Design Brief

> 狀態：**需求方已核可方向（2026-07-20）**——確認「拆標籤」與本文件 §2 的標籤列長相。
> 規劃：Claude（Opus 4.8）　基線：UX-PLAYER-IA1-DECISION（本卡修訂其 §1 凍結項）
> 前置：UX-PLAYER-SECTIONS1（🏁完成）、UX-PLAYER-FIELDVIZ1（🏁完成，已上線）

## 0. 為什麼要改凍結基線

UX-PLAYER-IA1 §1 凍結四層（總覽／打法·球路／分項與對戰／生涯），並以 role 切換鈕控制全頁身分。
上線後需求方走查提出三點：

1. **守備應與其他層同級**，不該埋在生涯層裡。
2. **「打法」應改為「打擊」**。
3. **雙棲球員應該「出現兩個標籤頁」**——即 role 不該是切換鈕，而應攤成標籤。

第 2 點單獨做會與 role 鈕的「打擊」撞名（同頁兩個「打擊」chip）；第 3 點正好一併解決——
role 鈕消失後就不存在撞名。故三點合併為一次 IA 修訂。

**本卡修訂 IA1 §1 的凍結項**，並回寫該決策文件；不是在實作卡內靜默改方向。

## 1. 修訂後的骨架

| 舊（IA1 凍結） | 新（本卡） |
|---|---|
| Hero＋role 切換鈕恆在 | Hero 恆在，**role 切換鈕移除** |
| 總覽常駐（依 role 顯示一種身分） | 總覽常駐，**雙棲時打擊／投球上下堆疊**（主身分在上） |
| L2 打法／球路（隨 role 改字） | **打擊**／**投球**，雙棲時**兩個都是標籤頁** |
| L3 分項與對戰 | 分項與對戰，**雙棲時兩種身分上下堆疊** |
| L4 生涯（含守備） | 生涯，**雙棲時上下堆疊**；**守備移出** |
| — | **守備**（新獨立層，與其他同級） |

**標籤列長相**

- 雙棲（例：余德龍）：`打擊｜投球｜分項與對戰｜守備｜生涯`
- 純打者（例：郭天信）：`打擊｜分項與對戰｜守備｜生涯`
- 純投手（例：羅戈）：`投球｜分項與對戰｜守備｜生涯`

四～五個標籤。**實作實測更正**：5 個標籤在 375px 原本溢出 11px（tablist 346 > 335），
最後一個標籤會被切掉一角；縮減標籤內距後 335＝335 剛好放下、完整可見、不需橫捲。
若日後再加第 6 個標籤，必然需要橫捲或改導覽形式——這是本結構的硬上限。

## 2. 為什麼堆疊而不是再拆更多標籤

若「分項與對戰」「生涯」也各自拆成打擊版與投球版，雙棲球員會有 7 個標籤（打擊｜投球｜打擊分項｜
投球分項｜守備｜打擊生涯｜投球生涯），375px 必然橫捲、且與藍圖 §3 的減法原則相衝。
堆疊讓「這個人兩種身分的表現」在同一頁可直接對照，也免去任何隱性 role 狀態。

## 3. 具體安置

| 層 | 內容 | 雙棲處理 |
|---|---|---|
| 總覽（常駐） | SeasonSection＋TraitsChips＋SeasonTrendCard | 兩身分堆疊，各自加身分小標 |
| 打擊 | TrackingSection＋QualitySection＋BattedMixSection（打者視角） | 獨立標籤，不含投球內容 |
| 投球 | 同上（投手視角）＋MovementSection | 獨立標籤 |
| 分項與對戰 | CareerTrendCard＋VsTeamCard＋SplitsSection | 兩身分堆疊 |
| **守備**（新） | FieldingSection（身分圖＋價值卡＋本季／生涯表） | 與 role 無關，不需堆疊 |
| 生涯 | CareerSummary＋CareerYearlySection＋SabrSection | 兩身分堆疊（SABR 本就分打者／投手） |

守備移出生涯層後，IA1 遷移 map #16「守備歸 L4 生涯」一併失效，改為獨立層。

## 4. URL 契約（不可破壞既有連結）

- `?sec=` 值域擴充為 `batting | pitching | splits | fielding | career`；舊值 `approach` **保留相容**，
  導向該球員的主身分內容頁（純投手→投球、其餘→打擊）。
- `?role=` 已上線於 production，移除會使既有分享連結失效。改為：**仍接受但不再控制全頁**，
  僅在未給 `?sec=` 時決定落哪個內容頁（`role=pitching` → 投球）。
- 預設層：退役／教練（`roster_level` 為 null）→ `career`；其餘 → 主身分內容頁。

## 5. 資料載入

現行依層延後載入的分組維持，並依新結構調整：

| 群組 | 需要它的層 |
|---|---|
| profile／season／advanced／trend | 常駐（總覽；雙棲需兩身分皆抓） |
| tracking／movement | 打擊／投球 |
| splits／vsTeam／careerMonthly | 分項與對戰（雙棲需兩身分） |
| **fielding／fieldingCareer** | **守備**（自生涯層移出） |
| career（逐年） | 生涯（雙棲需兩身分） |

雙棲球員在堆疊層需要兩種 role 的資料，請求量會增加——以既有 key 快取機制避免重複，
並維持「不因分層而重複請求同一端點」的驗收條件。

## 6. 一併處理的小修正

**守位身分圖只在多守位（≥2 守位）時渲染**。需求方指出單一守位時它等於重畫下方表格一列、
沒有增值；圖的價值是「守備範圍的組合」，只有多守位球員才存在。

> 此項原應於 UX-PLAYER-FIELDVIZ1 裁決當下實作，但當時該卡 worktree 已回收、編輯失敗，
> 只寫進文件未進程式；於本卡走查時以實測發現（單一守位的郭天信仍畫圖）並補實作。

## 7. 狀態契約（沿用 IA1 §1.2，調整處如下）

| 狀態 | 規則 |
|---|---|
| 結構性空 | 無該身分 → 該內容頁的標籤**不出現**（純打者無「投球」頁） |
| 資料空 | 堆疊層中某一身分無資料 → 該身分區塊顯示帶原因的空態，不整段消失 |
| 退役 | 預設落生涯層；總覽顯示既有的引導空態 |
| 守備層無資料 | 整層仍可及，內容顯示空態（不隱藏標籤，避免標籤數隨球員跳動） |
