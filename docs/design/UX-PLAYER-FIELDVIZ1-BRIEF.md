## 0. v1 為何作廢

v1 提案「守位出賽數泡泡圖」（圓點面積 ∝ 出賽數）經需求方走查後判定**無實際意義**——
它只是把表格裡的出賽數換一種畫法，看不出守備價值。v2 全案重做。

研究結論（見研究文件 §1、§4）：**球員級守備範圍／價值圖以現有資料無法誠實支撐**。
OAA 需守備員起始站位（永遠沒有）；UZR/DRS 分區法資料型態相符，但 2026 單季每隊每守位
常規先發僅 111–186 個機會，切 9 區後每區 12–21 球，是雜訊。故改採 **A＋C**：
A 給窄但真實的價值訊號，C 給身分描述且不假裝是價值。

## 1. 交付兩個元件（原置於 L4 生涯層守備區；**守備已由 UX-PLAYER-IA2 提升為獨立層**，見 §8）

### 元件 C：守位身分圖（不宣稱價值）

回答「他是什麼樣的守備者」，不回答「他守得好不好」。

| 決策 | 內容 |
|---|---|
| 形式 | 球場示意圖（複用 `components/spray-chart.tsx` 既有扇形幾何），9 守位固定座標 |
| 標注 | 守過的位置逐一標注**守備局數**（2018+）或**出賽數**（2018 前），數值直接寫在旁邊 |
| 多守位 | 依需求方指示：各守位**分別標注**，並在標題列加「多守位」標記 |
| 顏色 | **不以顏色深淺表示好壞**——這是身分圖不是價值圖，避免誤讀為守備評價 |
| DH | 不表達（守備資料本就不含 DH，見 memory `dh-position-inference`） |
| 二軍 | 排除，僅 `kind_code='A'`（需求方指示） |

### 元件 A：守備價值卡（依守位群分流，內容各不相同）

| 守位群 | 呈現 | 為何是這個 | 誠實邊界 |
|---|---|---|---|
| **外野**（LF/CF/RF） | 助殺 **A/9**＋累計 A，對照同守位聯盟中位數 | 外野助殺是官方計數中少數能反映「阻嚇型」守備價值者 | **阻嚇悖論**：助殺少可能因跑者不敢進壘＝守備價值高，指標方向與真實價值相反 |
| **內野**（2B/SS/3B） | **DP/9**、**TC/9**、守備率，對照同守位聯盟中位數 | 雙殺參與與守備機會密度是內野可得的最接近價值的訊號 | 受投手群滾飛傾向影響，非純個人能力 |
| **一壘** | 只列計數，**不做價值宣稱** | 一壘守備價值本就難以量化 | 坦白略過優於硬湊 |
| **捕手** | **CS%** 與 **RA/9**（沿用 SABR Phase A 既有 `catcher_runs`，已上線） | CS% 分母即機會數，是唯一天生乾淨的率 | RA/9 含非自責分、非 cERA（沿用既有標註） |
| **投手** | 不做 | 全聯盟 2026 僅 300 個守備事件，樣本不足 | — |

## 2. 機會數校正（需求方追問後定案，研究文件 §8）

1. **分母一律為守備局數**（`fielding_innings.outs / 3`），不是出賽數、更不是打席數。
   實測：同守位每場守備局數離散大（左外野 3.8–8.7），以出賽數為分母會使代守型球員率值灌水逾兩倍。
2. **率值僅 2018 年起**。官方 `fielding_seasons` 無局數欄位；1990–2017 **只顯示累計計數，不算率**。
3. **合格門檻 100 局**方可進入聯盟對照。實測 2025 各守位合格者 12–21 人（每隊 2–3.5 人），
   足以談中位數；未達門檻者顯示數值但不給對照，並標示樣本不足。
4. 每個率值旁必須附**局數**，讓讀者自行判斷樣本可信度。

## 3. 揭露文案位置（需求方採納建議）

| 類型 | 位置 | 理由 |
|---|---|---|
| **會直接導致誤讀**：外野助殺阻嚇悖論 | **指標旁 inline** | 讀者當下就必須看到，否則會把「助殺少」讀成「臂力差」 |
| 一般脈絡：投手群滾飛傾向、RA/9 非 cERA、率值 2018 起、合格門檻 | 頁尾資料說明 | 沿用現行球員頁的統一收納慣例 |

## 4. 需要的 API 變更（v1 誤判為純前端，此處更正）

現行 `/api/v1/players/{id}/fielding` 只回官方計數（`g/tc/po/a/e/dp/tp/pb/cs/sba/fpct`），
**不含守備局數與聯盟對照**。本卡需擴充：

- 回傳每守位的 `outs`（join `cpbl.fielding_innings`，2018+；更早為 null）
- 回傳同守位同年的聯盟對照值（中位數）與合格人數 `n`，供前端顯示對照與樣本揭露
- 唯讀查詢，`db_scope: read`；不新增表、不寫入

## 5. 狀態契約（沿用 IA1 §1.2）

| 狀態 | 規則 |
|---|---|
| 正常 | 依 §1 渲染 |
| 結構性空 | 無任何守備列（純 DH／無守備紀錄）→ 兩個元件皆不渲染，維持現有整區隱藏 |
| 資料空 | 有計數無局數（2018 前）→ 身分圖顯示出賽數、價值卡只列計數不列率 |
| 樣本不足 | 局數 < 100 → 顯示數值但不給聯盟對照，標示樣本不足 |
| 錯誤 | 守備請求失敗 → 兩元件不渲染，既有表格降級不受影響（模組獨立降級） |

## 6. 可及性

- SVG `role="img"`，`aria-label` 描述結論（例：「守位分布：游擊手 640 局、二壘手 120 局，多守位」）。
- 圖為輔助表達，所有數值在既有守備表與價值卡文字中皆可取得，不做只存在於圖中的資訊。
- 顏色不作為唯一區辨手段；身分圖本就不用顏色編碼價值。

## 7. 驗收

- [ ] 身分圖：多守位（余德龍）、純投手（羅戈）、退役（彭政閔）、無守備紀錄者 四情境正確；多守位標記出現。
- [ ] 價值卡：外野／內野／一壘／捕手 四種分流各自呈現正確內容；一壘不出現價值宣稱。
- [ ] 率值一律以局數為分母；2018 前無率值；未達 100 局無聯盟對照且標示樣本不足；每個率旁有局數。
- [ ] 阻嚇悖論揭露出現在指標旁，非僅頁尾。
- [ ] 二軍資料未進入任一元件。
- [ ] 375px 無水平溢出；`npm test`／`tsc`／`build:check`／`ruff`／`pytest` 全綠；新端點加 route 快照 EXPECTED。

## 8. 後續修訂（append-only，2026-07-20）

本卡交付並上線後，需求方走查提出兩點，由 [`UX-PLAYER-IA2-BRIEF.md`](UX-PLAYER-IA2-BRIEF.md) 承接：

1. **守備自生涯層移出，成為與打擊／投球／分項與對戰／生涯同級的獨立標籤層**。
   本文件 §1「皆置於 L4 生涯層守備區」與 IA1 遷移 map #16 隨之失效；兩個元件的內容與統計口徑不變。
2. **守位身分圖改為僅在多守位（≥2 守位）時渲染**——需求方指出單一守位時它等於重畫下方表格的一列、
   沒有增值；圖的價值是「守備範圍的組合」，只有多守位球員才存在。
   本文件 §1 元件 C 原未設此條件，實作亦未落實（該修改當時因 worktree 已回收而編輯失敗，
   只寫進 IA2 Brief 未進程式），已於 UX-PLAYER-IA2 補實作。

§2 機會數校正、§3 揭露位置、§4 API 變更、§5 狀態契約、§6 可及性均維持有效。
