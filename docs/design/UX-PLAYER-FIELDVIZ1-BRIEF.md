# UX-PLAYER-FIELDVIZ1 守備守位分布圖 Design Brief

> 狀態：**待需求方核可**
> 日期：2026-07-20　規劃：Claude（Opus 4.8）　基線：PRODUCT_UX_BLUEPRINT v0.2＋UX-PLAYER-IA1 §1（四層骨架凍結）

## 1. 要解的問題

現況守備區只有數字表（守位／出賽／TC／PO／A／E／DP／守備率）。表格能查得精確，但回答不了兩個一眼就該看出的問題：

- **他主要守哪裡？** 現在要比對各列的出賽數才知道。
- **他守備範圍多廣？** 兼守三個內野位置與只守一壘，在表上長得一樣，只有列數不同。

守備是球員身分的一部分（藍圖 §5.7 將守備歸生涯層），但目前它是全頁最「純表格」的區塊。

## 2. 資料事實（已查證，非估計）

| 項目 | 事實 |
|---|---|
| 來源 | 現有 `/api/v1/players/{id}/fielding?scope=season\|career`，**無需 API 變更** |
| 守位值 | 9 種中文名：投手／捕手／一壘手／二壘手／三壘手／游擊手／左外野手／中外野手／右外野手（`cpbl.fielding_current` 實查） |
| 指定打擊 | **不在守備資料中**（DH 須由「打擊出賽−守備出賽」推算，見既有主守位邏輯）——本圖不表達 DH |
| 每列欄位 | `pos, g, tc, po, a, e, dp, tp, fpct`（捕手另有 `pb, cs, sba`） |
| 可複用幾何 | `components/spray-chart.tsx` 已有球場扇形座標系（W=300 H=232、本壘貼底、牆半徑餘弦內插），本圖沿用同一視覺語彙 |

## 3. 視覺規格

**一張圖，九個固定座標點**，疊在既有兩張表之上。

| 決策 | 內容 | 為什麼 |
|---|---|---|
| 尺寸編碼 | 圓點**面積** ∝ 該守位出賽數 `g` | 用面積不用半徑，半徑映射會把差距視覺放大約平方倍 |
| 雙尺度 | **實心＝生涯累計**，**外圈環＝本季** | 一圖同時回答「以前守哪」與「今年守哪」，且**不新增切換**——SECTIONS1 剛移除守備區的本季／生涯切換，不可再引入 |
| 主守位 | 出賽數最高者以 accent 色與標籤強調，其餘中性色 | 「主要守哪裡」是第一問，應由視覺直接回答 |
| 最小可視 | 出賽 1–2 場的兼守仍畫，設最小半徑 | 兼守紀錄本身就是資訊，不可因小而消失 |
| 互動 | hover／focus 顯示該守位的 G、守備率、失誤 | 精確值仍以下方表格為準，圖不取代表 |

**狀態契約**（沿用 IA1 §1.2）

| 狀態 | 規則 |
|---|---|
| 正常 | 依上表渲染 |
| 結構性空 | 無任何守備列（純 DH／退役無守備紀錄）→ **不渲染圖**，維持現有「整區不顯示」邏輯 |
| 資料空 | 有生涯無本季（退役）→ 只畫實心，不畫外圈；不顯示空的本季圖例 |
| 錯誤 | 守備請求失敗 → 圖不渲染，表區既有降級不受影響（模組獨立降級） |

## 4. 可及性

- SVG `role="img"`，`aria-label` 以文字描述結論，例：「守位分布：主要守游擊手（本季 87 場），另兼守二壘手、三壘手」。
- 圖為**輔助表達**，所有數值在下方表格皆有；不做只存在於圖中的資訊，故不需要為圖另建鍵盤操作。
- 顏色不作為唯一區辨手段：主守位另有文字標籤。

## 5. 驗收情境

| 情境 | fixture | 預期 |
|---|---|---|
| 多守位 | 余德龍 `0000003563` | 多個圓點、主守位標示、本季外圈與生涯實心並存 |
| 純投手 | 羅戈 `0000005151` | 僅投手位一點 |
| 退役 | 彭政閔 `0000000667` | 僅生涯實心、無外圈 |
| 無守備 | 純 DH／無紀錄者 | 整區不渲染圖 |

375 px 無水平溢出；`tsc`、`build:check`、`npm test` 全綠。
