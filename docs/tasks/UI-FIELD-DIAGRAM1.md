## 問題

現行守位身分圖（UX-PLAYER-FIELDVIZ1 交付、UX-PLAYER-IA2 沿用）以**真實球場座標**擺放守位，
導致內野守位在小尺寸下彼此過近，標籤與局數文字重疊。實測（dev :3016，SVG `getBBox` 相交檢測）：

| 情境 | 守位數 | 重疊 |
|---|---|---|
| 余德龍（游擊／三壘／投手） | 3 | 1 組（「45 局」↔「三壘」） |
| 高捷（一壘／二壘／三壘／左外野／右外野） | 5 | 1 組（「12 場」↔「一壘」） |

該圖**只在多守位時渲染**，因此上述是它的全部使用情境——等於每個渲染案例都有重疊。

更根本的問題：這張圖**本來就不宣稱空間精確度**（它是身分圖，不是守備範圍／價值圖，
見 [`UX-PLAYER-FIELDVIZ1_RESEARCH.md`](../research/UX-PLAYER-FIELDVIZ1_RESEARCH.md) §1）。
既然不需要真實座標，就沒有理由付出可讀性代價。改採轉播記分板慣用的**制式排版**：
守位固定格位、標籤置於方框、彼此不相鄰，可讀性優先。

## 目標

- [ ] 產出**共用元件**（置於 `web/src/components/`，非球員頁私有），props 為「守位 → 顯示內容」的映射，
      不假設資料來源；球員頁與賽況頁共用同一元件。
- [ ] 排版採轉播風制式格位（外野三格一列、內野四格、投手、捕手），**非真實座標**；
      任兩個文字框不得相交（以 `getBBox` 相交檢測為驗收，非目視）。
- [ ] 每格可帶主標（守位碼／中文名）與副標（自由字串：局數／場數／球員名），由呼叫端決定。
- [ ] 支援「未使用格位」的低調呈現，讓讀者看得出哪些守位沒有資料。
- [ ] 接上球員頁：取代現行守位身分圖的排版，內容與語意不變（仍只在多守位時渲染、不以顏色編碼好壞）。

賽況頁的**先發守位**與**守備異動**。資料可行性已查證：
