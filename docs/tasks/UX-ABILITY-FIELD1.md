## 問題

現行守備軸（[`ability.py`](../../src/cpbl/api/routers/ability.py) 的 `fld` CTE）：

- 非捕手：`RF = (PO + A) / G`，再除以「該守位×該年」聯盟均值正規化 → PR
- 捕手：改用阻殺率 `CS%`（分母本身即機會數，不受影響）

**助殺已含在 RF 內**（`PO + A`），無須另加軸；守位內正規化也已正確處理跨守位不可比的問題。
真正的缺陷在**分母是「每場」而非「每 9 局」**：同守位每場守備局數散布 3.8–8.7 局
（研究文件 §8.1 實測），代守型球員每場機會天生較少，RF/G 因而混入「每場守幾局」的訊號。

## 已取得的證據（2026 一軍，出賽 ≥8 場）

RF/G 與 RF/9 的球員排序相關性：

| 守位 | 相關 | 人數 |
|---|---|---|
| 游擊手 | **−0.623** | 13 |
| 投手 | 0.070 | 92 |
| 三壘手 | 0.116 | 15 |
| 一壘手 | 0.211 | 24 |
| 捕手 | 0.245 | 16 |
| 右外野 | 0.264 | 21 |
| 中外野 | 0.628 | 18 |
| 二壘手 | 0.673 | 21 |
| 左外野 | 0.732 | 20 |

個案佐證：某游擊手 RF/G 2.78（全體最低）但每場僅守 6.0 局，換算 RF/9 為 4.19，與他人差距大幅收斂。

**誠實界線**：各守位樣本僅 13–24 人（投手除外），相關係數雜訊大；游擊手的負相關**不可**
單獨當作結論，只能說「兩種分母給出的排序在多數守位上幾乎無關」。
