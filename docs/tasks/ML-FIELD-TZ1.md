## 背景

現行能力卡守備軸 `RF = (PO+A)/G` 衡量「經手多少球」而非守備價值；改分母只修正時間基準，
不改變衡量對象（見 [`UX-ABILITY-FIELD1_PHASE1.md`](../research/UX-ABILITY-FIELD1_PHASE1.md)，
該卡 Phase 2 已依需求方裁決不執行）。

國外對「無座標資料」的處境有成熟解法：[Total Zone](https://www.baseball-reference.com/about/total_zone.shtml)
只需逐打席的「誰處理的」，估算超越期望的出局數。我們的 `game_livelog` 2018–2026 有約
**13.6 萬筆**可抽出守位的事件（逐球追蹤僅 5,998 筆），且可跨季累積以滿足
[UZR 文件](https://library.fangraphs.com/defense/uzr/)明訂的「三年才可下結論」門檻。

## 研究問題（本卡要回答的）

- [x] **文字解析涵蓋率**：**95.31%**（138,370／145,178 擊球入場），九季穩定 94.88–96.26% 無斷層。殘餘 4.69% **有系統性偏誤**：犧牲觸擊 2,159 筆全數落空（「擊出球(觸擊)」無方向詞），系統性低估投／捕／一／三壘手機會。詳 [報告](../research/ML_FIELD_TZ1_FEASIBILITY.md) §1。
- [x] **期望值模型怎麼建**：**此處否決**。`content` 的「方向」不是打擊落點而是處理結果（內野滾地球出局率 89.3% vs 滾出外野 0.2–0.5%），以此為條件屬循環論證；且歸屬不對稱——守備成功具名 30,277 筆、失敗僅 10–17／5,000 筆，內野失敗只知左／中／右側，游擊與三壘責任無法分離（欠定方程組）。詳 §2。
- [x] **跨季累積與年代校正**：**可比，且校正需求輕**。聯盟出局率九季 +6.9pt，但完全由擊球型態組成解釋（平飛 17.4%→10.1%、高飛 36.7%→46.5%）；類別內部無漂移（外野高飛球出局率 68.11%→68.98% 平坦），故只需條件化於擊球型態，不需比照 RE 矩陣全面重估。詳 §3。
- [x] **驗證方案**：已執行場次數對帳（逐年與 spec 基線 §3 完全一致）與每場打席數合理性（75.5–81.6，2018–20 偏高顯示切界過度切分）。若實作建議三階：出局守恆 ±3%、與官方守備率中度正相關、金手套僅作弱訊號不可當 ground truth。詳 §4。
- [x] **規模與風險評估**：發現未預見前置——**無逐場守備陣容**（`game_detail` 無先發、`fielding_innings` 僅球季層級），須由打者列 `defend_station_code` ＋換防事件重建，屬獨立子卡（不需新爬蟲）。外野三季累積約 885 次機會、精度僅 ±14 個機會。核心風險「看似精確實則錯誤」**無有效控制手段**，故建議否決而非緩解。詳 §5。

## 研究結論（執行者）

**完整逐守位 Total Zone：建議 NO-GO**——否決點為結構性歸屬不對稱，非樣本量或工程量，加大投入不會改善。
**不建議以 TZ 取代能力卡守備軸**：即使只做外野，內野手仍無值，混用會產生跨守位不可比的軸，比現況更糟。
唯一誠實選項為獨立的「外野守備範圍」探索性指標（非能力卡軸，規模 M，須明示精度限制），屬另卡並須需求方核可。
